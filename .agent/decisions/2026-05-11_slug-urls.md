title: Artifact viewer URLs use slug not UUID
choice: /a/:slug throughout the SPA; backend resolves slug OR UUID via byIdOrSlug()
scope: frontend routing, backend artifact routes
---

## Decision

All artifact links in the SPA navigate to `/a/<slug>` (e.g. `/a/the-unreasonable-effectiveness-of-html`) instead of `/a/<uuid>`. The backend `byIdOrSlug()` helper accepts either form so old UUID links don't break.

## Why

UUIDs are opaque and ugly in the address bar. Slugs are human-readable, shareable, and stable (slugs are immutable after creation). The owner will share these URLs; they should be readable.

## Tradeoffs

- Slug collisions are handled at creation time via `uniqueSlug()` — no runtime issue.
- If an artifact is fetched by slug and the slug matches multiple deleted+active rows (impossible by current uniqueness constraint), the `AND deleted_at IS NULL` clause ensures only the live one is returned.
- Existing UUID-based bookmarks still work because the backend resolves both.
