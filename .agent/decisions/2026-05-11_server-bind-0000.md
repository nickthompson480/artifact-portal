## Decision

Changed `app.listen(PORT, '127.0.0.1')` to `app.listen(PORT, '0.0.0.0')` so the portal is accessible from other devices on the local network (phone, iPad, other Mac).

## Why

The portal is a personal tool used on the local network. LAN access is a core use case — sharing artifacts with a phone or reading on an iPad. Localhost-only binding blocked this.

## Tradeoffs

No meaningful security regression: the Mac's firewall and home network provide the boundary. The portal requires auth (JWT cookie) for all owner routes regardless of bind address.
